# dbt-doc-to-pdf 導入手順

dbt の `manifest.json` / `catalog.json` を読み込み、PDF データカタログを生成する Python CLI ツールです。

---

## 1. 前提環境

### 必須要件

| 項目 | バージョン |
|------|-----------|
| Python | 3.13 以上 |
| uv | 最新版 |

### NixOS / Nix を使う場合

`nix-shell` を使って一時的に Python 3.13 と uv を利用できます。プロジェクトルートに以下の `shell.nix` を作成してください。

```nix
# shell.nix
{ pkgs ? import <nixpkgs> {} }:

pkgs.mkShell {
  buildInputs = [
    pkgs.python313
    pkgs.uv
  ];

  shellHook = ''
    echo "Python $(python3 --version)"
    echo "uv $(uv --version)"
  '';
}
```

起動方法:

```bash
nix-shell
```

Nix Flakes を使う場合は `flake.nix` の `devShells` に `pkgs.python313` と `pkgs.uv` を追加してください。

> **NixOS 固有の注意:** dbt-duckdb は DuckDB の共有ライブラリ (`libstdc++.so.6`) を必要とします。リポジトリに含まれる `dbt.sh` スクリプトが Nix ストアから自動的に `LD_LIBRARY_PATH` を設定するため、dbt コマンドはこのスクリプト経由で実行してください。

## 2. uv のインストール

### 公式インストーラー（Linux / macOS）

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

インストール後、シェルを再起動するか以下を実行してパスを通します:

```bash
source $HOME/.local/bin/env   # bash / zsh の場合
fish_add_path $HOME/.local/bin  # fish の場合
```

### Nix / NixOS でのインストール

```bash
# nixpkgs から直接インストール
nix profile install nixpkgs#uv

# または nix-env
nix-env -iA nixpkgs.uv
```

バージョン確認:

```bash
uv --version
```

## 3. プロジェクトのセットアップ

リポジトリをクローン後、`uv sync` で仮想環境の作成と依存パッケージのインストールを一括実行します。

```bash
git clone <repository-url>
cd dbt-doc-to-pdf

# 仮想環境の作成 + 依存パッケージのインストール
uv sync
```

`uv sync` は `uv.lock` に記録された固定バージョンを使ってインストールするため、環境の再現性が保証されます。仮想環境は `.venv/` に作成されます。

---

## 4. インストール済み依存パッケージ一覧

`pyproject.toml` で定義されている依存パッケージは以下のとおりです。

| パッケージ | バージョン制約 | 用途 |
|-----------|--------------|------|
| `dbt-core` | `>=1.11.8` | dbt コアライブラリ（manifest / catalog のスキーマ解釈） |
| `dbt-duckdb` | `>=1.10.1` | DuckDB アダプタ（サンプルプロジェクトの実行） |
| `reportlab` | `>=4.4.10` | PDF 生成エンジン |
| `pdfplumber` | `>=0.11.9` | PDF 内容の検証（テスト用） |
| `pytest` | `>=9.0.3` | テストフレームワーク |

実際にインストールされたバージョンは `uv.lock` または以下のコマンドで確認できます:

```bash
uv pip list
```

## 5. ツールの実行方法

### 基本コマンド

```bash
uv run dbt-doc-to-pdf [オプション]
```

### オプション一覧

| オプション | デフォルト値 | 説明 |
|-----------|------------|------|
| `--target-dir PATH` | `./target` | dbt の `target/` ディレクトリのパス（`manifest.json` と `catalog.json` が格納されている場所） |
| `--output PATH` | `./catalog.pdf` | 出力する PDF ファイルのパス |
| `--project NAME` | `sample_project` | PDF 表紙に表示するプロジェクト名 |

### 使用例

```bash
# サンプルプロジェクトの target/ から PDF を生成（デフォルト設定）
uv run dbt-doc-to-pdf \
  --target-dir sample_project/target \
  --output my_catalog.pdf \
  --project "My dbt Project"

# ヘルプを表示
uv run dbt-doc-to-pdf --help
```

実行すると検出したモデル一覧と列数が標準出力に表示され、PDF が生成されます:

```
モデル数: 5
  [staging     ] stg_orders (8 columns)
  [ecommerce   ] fct_orders (12 columns)
  ...

PDF を生成しました: my_catalog.pdf
```

## 6. テスト実行方法

テストは `sample_project/target/` に格納済みの `manifest.json` / `catalog.json` を使用します。

### 全テストを実行

```bash
uv run pytest
```

### 特定のテストファイルを実行

```bash
uv run pytest tests/test_extractor.py
uv run pytest tests/test_pdf_content.py
```

### 特定のテスト関数を実行

```bash
uv run pytest tests/test_extractor.py::test_extract_models_count -v
```

### テスト構成

| ファイル | 内容 |
|---------|------|
| `tests/conftest.py` | フィクスチャ定義（manifest / catalog の読み込み、PDF 生成） |
| `tests/test_extractor.py` | モデル抽出ロジックのテスト |
| `tests/test_pdf_content.py` | 生成 PDF の内容検証テスト |

## 7. dbt サンプルプロジェクトの使い方

`sample_project/` は DuckDB をバックエンドとした ecommerce サンプルプロジェクトです。`target/` ディレクトリを再生成する手順を以下に示します。

### 前提

サンプルプロジェクトのディレクトリ構成:

```
sample_project/
├── dbt_project.yml
├── profiles.yml          # DuckDB 接続設定（ローカルファイル dev.duckdb を使用）
├── models/
│   ├── staging/          # ビューとしてマテリアライズ
│   └── ecommerce/        # テーブルとしてマテリアライズ
└── target/               # 生成物（manifest.json, catalog.json など）
```

### NixOS の場合（推奨）: `dbt.sh` を使う

リポジトリルートの `dbt.sh` スクリプトは NixOS 向けの `LD_LIBRARY_PATH` 設定を自動的に行います。

```bash
# モデルのビルド
./dbt.sh run

# テストの実行
./dbt.sh test

# ドキュメント生成（manifest.json + catalog.json を再生成）
./dbt.sh docs generate
```

### 通常環境の場合: uv run dbt を使う

```bash
cd sample_project

# モデルのビルド
uv run dbt run --profiles-dir .

# テストの実行
uv run dbt test --profiles-dir .

# ドキュメント生成（target/ に manifest.json と catalog.json を出力）
uv run dbt docs generate --profiles-dir .
```

### `target/` 再生成後に PDF を作成

```bash
# プロジェクトルートに戻る
cd /path/to/dbt-doc-to-pdf

# PDF を生成
uv run dbt-doc-to-pdf \
  --target-dir sample_project/target \
  --output catalog.pdf \
  --project sample_project
```

### DuckDB パスのカスタマイズ

`profiles.yml` は環境変数 `DBT_DUCKDB_PATH` でデータベースファイルのパスを変更できます:

```bash
DBT_DUCKDB_PATH=/tmp/mydb.duckdb ./dbt.sh run
```

## 8. Docker での使い方

Python / uv 環境を用意せずに Docker だけで PDF を生成できます。

### 前提条件

- Docker Engine（または Docker Desktop）
- Docker Compose v2 以上

### 手順

#### 1. 自分の dbt プロジェクトで `dbt docs generate` を実行する

コンテナは `manifest.json` と `catalog.json` が存在する target ディレクトリを読み込むだけです。  
事前にホスト側でドキュメント生成を完了させてください。

```bash
# 例: 自分の dbt プロジェクトで実行
cd /path/to/your-dbt-project
dbt docs generate
# → target/manifest.json, target/catalog.json が生成される
```

#### 2. docker compose up を実行する

```bash
cd /path/to/dbt-doc-to-pdf

# 自分のプロジェクトを指定する場合
TARGET_DIR=/path/to/your-dbt-project/target \
PROJECT=your_project_name \
docker compose up

# サンプルプロジェクト（./sample_project/target）を使う場合はそのまま実行
docker compose up
```

#### 3. 生成された PDF を確認する

```bash
ls ./output/catalog.pdf
```

`./output/` ディレクトリに `catalog.pdf` が出力されます。

---

### 環境変数一覧

| 変数名 | デフォルト値 | 説明 |
|--------|------------|------|
| `TARGET_DIR` | `./sample_project/target` | ホスト側の target ディレクトリのパス |
| `PROJECT` | `sample_project` | PDF 表紙に表示するプロジェクト名 |
| `OUTPUT` | `/output/catalog.pdf` | コンテナ内での出力先パス（基本変更不要） |

### `.env` ファイルを使う場合

毎回環境変数を指定するのが手間な場合、プロジェクトルートに `.env` ファイルを作成してください。

```bash
# .env
TARGET_DIR=/path/to/your-dbt-project/target
PROJECT=your_project_name
```

```bash
docker compose up
```

## トラブルシューティング

### `libstdc++.so.6` が見つからない（NixOS）

dbt-duckdb 実行時に以下のようなエラーが出る場合:

```
OSError: libstdc++.so.6: cannot open shared object file
```

`dbt.sh` を経由して実行してください。直接 `uv run dbt` を使う場合は、手動で `LD_LIBRARY_PATH` を設定します:

```bash
LIBSTDCXX=$(find /nix/store -maxdepth 3 -name "libstdc++.so.6" 2>/dev/null | grep "gcc-14" | head -1 | xargs dirname)
export LD_LIBRARY_PATH="$LIBSTDCXX${LD_LIBRARY_PATH:+:$LD_LIBRARY_PATH}"
```

### `target` ディレクトリが見つからない

```
ERROR: target ディレクトリが見つかりません: target
```

`--target-dir` オプションで正しいパスを指定するか、先に `dbt docs generate` を実行して `target/` を生成してください。
